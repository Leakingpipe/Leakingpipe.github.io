Concepts review: Rasterization

This is a serie where I review some old concepts I’ve learned a long time ago.

Context

When all the models are compressed to a Normalized Device Coordinates, we need viewport transform to project the coordinate system to screen space.

Sampling & Inside test

Inside test is to check whether a center point of a pixel is inside of a triangle. How can we do this? Using the Cross Product
For a triangle △ABC\triangle ABC△ABC and a pixel P(x, y), we can use cross product to tell whether P is in △ABC\triangle ABC△ABC
In a 2D plane, the result of cross production of vector a⃗\vec{a}a and b⃗\vec{b}b is :

a⃗×b⃗=xayb−xbya\vec{a} \times \vec{b} = x_a y_b - x_b y_a a×b=xa​yb​−xb​ya​

if its >0, then b⃗\vec{b}b is on the left side of a⃗\vec{a}a
if its <0, then b⃗\vec{b}b is on the right side of a⃗\vec{a}a

Bounding box

It would be inefficient to do cross production on all the pixels on the screen
