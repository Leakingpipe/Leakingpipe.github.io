Concepts review: Rasterization

This is a serie where I review some old concepts I’ve learned a long time ago.

¶Context

When all the models are compressed to a Normalized Device Coordinates, we need viewport transform to project the coordinate system to screen space.

¶Sampling & Inside test

Inside test is to check whether a center point of a pixel is inside of a triangle. How can we do this? Using the Cross Product
For a triangle △ABC\triangle ABC△ABC and a pixel P(x, y), we can use cross product to tell whether P is in △ABC\triangle ABC△ABC
In a 2D plane, the result of cross production of vector a⃗\vec{a}a and b⃗\vec{b}b is:

a⃗×b⃗=xayb−xbya\vec{a} \times \vec{b} = x_a y_b - x_b y_a a×b=xa​yb​−xb​ya​

if its >0, then b⃗\vec{b}b is on the left side of a⃗\vec{a}a
if its <0, then b⃗\vec{b}b is on the right side of a⃗\vec{a}a

Edge Function
Let’s define an edge function E01(P)E_{01}{(P)}E01​(P) to test whether P is on a certain size of edge V0V1V_0V_1V0​V1​ :

E01(x,y)=(P−V0)×(V1−V0)E_{01}(x, y) = (P - V_0) \times (V_1 - V_0) E01​(x,y)=(P−V0​)×(V1​−V0​)

and the vector coordinates:V0=(x0,y0),V1=(x1,y1),P=(x,y)V_0=(x_0, y_0), V_1=(x_1, y_1), P=(x, y)V0​=(x0​,y0​),V1​=(x1​,y1​),P=(x,y)

V0P⃗=(x−x0,y−y0)\vec{V_0P} = (x - x_0, y - y_0) V0​P​=(x−x0​,y−y0​)

V0V1⃗=(x1−x0,y1−y0)\vec{V_0V_1} = (x_1 - x_0, y_1 - y_0) V0​V1​​=(x1​−x0​,y1​−y0​)

In order to fit the form of Ax+By+CAx+By+CAx+By+C , we usually calculate V0V1⃗×V0P⃗\vec{V_0V_1} \times \vec{V_0P}V0​V1​​×V0​P​ :

E(x,y)=(x−x0)(y1−y0)−(y−y0)(x1−x0)E(x, y) = (x - x_0)(y_1 - y_0) - (y - y_0)(x_1 - x_0) E(x,y)=(x−x0​)(y1​−y0​)−(y−y0​)(x1​−x0​)

If all three edges all satisfy E(x,y)≥0E(x, y) \geq 0E(x,y)≥0, then this pixel is inside the triangle.

¶Bounding box

It would be inefficient to do cross production on all the pixels on the screen, so let’s just create a bounding box so that We only iterate through the smallest rectangular region that encloses the triangle.

xmin=min⁡(xA,xB,xC),xmax=max⁡(xA,xB,xC)x_{min} = \min(x_A, x_B, x_C), \quad x_{max} = \max(x_A, x_B, x_C) xmin​=min(xA​,xB​,xC​),xmax​=max(xA​,xB​,xC​)

ymin=min⁡(yA,yB,yC),ymax=max⁡(yA,yB,yC)y_{min} = \min(y_A, y_B, y_C), \quad y_{max} = \max(y_A, y_B, y_C) ymin​=min(yA​,yB​,yC​),ymax​=max(yA​,yB​,yC​)

¶Barycentric Coordinates & Interpolation

Once we are sure that the pixel is inside of the triangle, we have to calculate its Z-depth, color, normal and UV. These attribute can get from vertex A, B, C, however the property of the pixel inside need to get by interpolating, that’s when we introduce Barycentric Coordinates.

definition:
For any P in the triangle plane can be represented as linear combination of those three vertices of the triangle:

P=αA+βB+γCP = \alpha A + \beta B + \gamma C P=αA+βB+γC

and satisvy the condition:

α+β+γ=1\alpha + \beta + \gamma = 1 α+β+γ=1

(α,β,γ)(\alpha, \beta, \gamma)(α,β,γ) is the barycentric coordinate of P, so if P is inside of the triangle, α,β,γ\alpha, \beta, \gammaα,β,γ should all be none negative.

How did we get to this conclusion?
barycentric coordinate can get from the area size of sub triangle

Let the total area be SABCS_{ABC}SABC​, and point PPP divides the triangle into △PBC\triangle{PBC}△PBC, △APC\triangle{APC}△APC, △ABP\triangle{ABP}△ABP.

α=SPBCSABC,β=SAPCSABC,γ=SABPSABC\alpha = \frac{S_{PBC}}{S_{ABC}}, \quad \beta = \frac{S_{APC}}{S_{ABC}}, \quad \gamma = \frac{S_{ABP}}{S_{ABC}} α=SABC​SPBC​​,β=SABC​SAPC​​,γ=SABC​SABP​​

As we know the magnitude of the cross product of 2d vector uuu and vvv is equal to the area of ​​the parallelogram they form, and the area of ​​the triangle is half of that SABC=12∥AB⃗×AC⃗∥S_{ABC} = \frac{1}{2} \| \vec{AB} \times \vec{AC} \|SABC​=21​∥AB×AC∥

Bringing in the coordinate calculation(here is actually the value of the previous edge function, divided by the total area):

α=−(x−xB)(yC−yB)+(y−yB)(xC−xB)−(xA−xB)(yC−yB)+(yA−yB)(xC−xB)\alpha = \frac{-(x - x_B)(y_C - y_B) + (y - y_B)(x_C - x_B)}{-(x_A - x_B)(y_C - y_B) + (y_A - y_B)(x_C - x_B)} α=−(xA​−xB​)(yC​−yB​)+(yA​−yB​)(xC​−xB​)−(x−xB​)(yC​−yB​)+(y−yB​)(xC​−xB​)​

β=−(x−xC)(yA−yC)+(y−yC)(xA−xC)−(xB−xC)(yA−yC)+(yB−yC)(xA−xC)\beta = \frac{-(x - x_C)(y_A - y_C) + (y - y_C)(x_A - x_C)}{-(x_B - x_C)(y_A - y_C) + (y_B - y_C)(x_A - x_C)} β=−(xB​−xC​)(yA​−yC​)+(yB​−yC​)(xA​−xC​)−(x−xC​)(yA​−yC​)+(y−yC​)(xA​−xC​)​

γ=1−α−β\gamma = 1 - \alpha - \beta γ=1−α−β

Once we get (α,β,γ)(\alpha, \beta, \gamma)(α,β,γ) , any property III inside of the triangle can be calculated through interpolation:

IP=αIA+βIB+γICI_P = \alpha I_A + \beta I_B + \gamma I_C IP​=αIA​+βIB​+γIC​

Note (correct perspective):
Linear interpolating depth ZZZ or texture coordinates UVUVUV in screen space is not allowed, because perspective projection is non-linear.
The correct approach is:

  1. first interpolate 1/w1/w1/w (www is the component in homogeneous coordinates).
  2. Then interpolate I/wI/wI/w.
  3. Finally, restore the attribute: IP=(αIAwA+βIBwB+γICwC)/(α1wA+β1wB+γ1wC)I_P = (\alpha \frac{I_A}{w_A} + \beta \frac{I_B}{w_B} + \gamma \frac{I_C}{w_C}) / (\alpha \frac{1}{w_A} + \beta \frac{1}{w_B} + \gamma \frac{1}{w_C})IP​=(αwA​IA​​+βwB​IB​​+γwC​IC​​)/(αwA​1​+βwB​1​+γwC​1​)

¶Z-Buffering

When 3D space is “flattened” into 2D, it can cause occlusion problems, that’s when we introduce Z-Buffer algorithm to solve this problem.

Z-Buffer: Stores the shallowest (nearest) depth value of each pixel on the current screen (initialized to infinity).
Frame Buffer: Stores color value.

For each pixel in each triangle:

  1. Getting the zcurrentz_{current}zcurrent​ using barycentric coordinates.
  2. Comparing zcurrentz_{current}zcurrent​ to the zbuffer[x,y]=colorz_{buffer[x,y]} = colorzbuffer[x,y]​=color in Z-Buffer.
  3. If zcurrent<zbuffer[x,y]z_{current} < z_{buffer}[x, y]zcurrent​<zbuffer​[x,y], which means this pixel is closer to the camera, then:
    • update depth: zbuffer[x,y]=zcurrentz_{buffer}[x, y] = z_{current}zbuffer​[x,y]=zcurrent​
    • update color: frame_buffer[x,y]=colorframe\_buffer[x, y] = colorframe_buffer[x,y]=color
  4. Else: ignore this pixel since it’s blocked.
