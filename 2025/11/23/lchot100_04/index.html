560. 和为 K 的子数组

利用前缀和和哈希表结合的方法。因为前缀和指的是从数组第1个元素一只加到第i个元素的和。它求出任意子数组的和的时间复杂度仅为O(1)

这里需要解释一下为何不使用滑动窗口，因为滑动窗口适用于没有负数的求和，或者求“无重复”的问题。

239. 滑动窗口最大值

这道题如果使用暴力解法，每次滑动都遍历窗口找最导致，时间复杂度就是$O(n*k)$
这里最好使用单调队列来实现对最大值的查找

不过我看题解的时候有个问题，就是怎样处理第k个数进来的那一刻保存数据。
这时候可以用i当做索引，当i达到k-1的时候，这时就是一个完整的窗口了，这个时候就记录结果，把队头放入result里面

76. 最小覆盖子串

我最开始的解法非常暴力，我打算用一个list去囊括所有的字母，但是这样时间复杂度会非常高。所以换一个优化版本的思路，用哈希表+计数器的方式。
首先可以把t想作一个“欠条”，可以统计t中每个字符出现了几次，并且用一个变量来记录，需要凑齐多少字符，才能还清这个“欠条”。
用两个指针，右指针用于扩张，不断将字符加入窗口。左指针用于收缩，即一旦窗口内包含了t的所有字符，就停下右指针，移动左指针。（这个做法是尝试让窗口左边和t里面不符合的字符丢掉，从而找到最优解。）当窗口不再达标，比如丢掉了一个t内的字符，就停止收缩，重新开始扩张。

不过俺在草稿纸上写了一下，我在想能否用一个字典来维护t的数量。比如t = {“A” : 1, “B” : 1, “C” : 1}，当扫描到一个A的时候，那么t[“A”] - 1，就变成t = {“A” : 0, “B” : 1, “C” : 1}。反正要求每个键后面的值<=0?(>0就是没有用完，说明窗口里面没有包含所有t的内容)
